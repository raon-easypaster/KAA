1 1장. 랜딩페이지 이해하기

1-1. 랜딩페이지란?

"특정 목적(신청, 후원, 안내 등)을 위해 제작된 단일 스크롤 웹페이지"

기관 공식 홈페이지와의 차이점

구분 기관 공식 홈페이지 랜딩페이지 (캠페인/사업용)
목적 복지관 전체 사업 및 정보 제공 한 가지 명확한 행동 유도
구조 복잡한 메뉴 중심 하나의 스토리 라인 (스크롤)
대상 불특정 다수 탐색자 특정 타겟 (예: 후원자, 참여자)
핵심 기능 정보 제공 및 연혁 안내 신청 / 후원 / 서명 참여

온라인은 또 다른 복지 서비스의 장입니다.
랜딩페이지는 클라이언트와 후원자를 가장 먼저 맞이하는 온라인 안내 데스크입니다.

2 2장. 랜딩페이지 구조 설계

기본 구조 5단계 (사회복지 마케팅 적용)

1

Hero (메인)

핵심 가치와 첫인상

2

문제 제기

대상자의 욕구/사회적 필요

3

해결 제시

프로그램/사업 내용 안내

4

신뢰 요소

투명성, 후기, 성과 지표

5

CTA

신청/후원/봉사 버튼

“기본 구조 5단계”는 방문자(참여자, 후원자)가 자연스럽게 스크롤을 내리며 행동을 결심하게 만드는 흐름입니다. 핵심은 방문자의 머릿속 질문 순서대로 정보를 제공하는 것입니다.

1 Hero: 첫 화면 핵심 메시지

🎯 역할: 3초 안에 "이 캠페인(프로그램)이 나에게 왜 의미가 있는가?"를 파악하게 합니다.
💡 사회복지 현장 예시 문구
  • 헤드라인: “발달장애 청년들의 자립을 응원하는 커피 한 잔의 기적”
  • 서브헤드: “바리스타 직무훈련생 모집 및 장학금 후원 캠페인”
  • 시각자료: 훈련생이 밝게 웃으며 커피를 내리는 고화질 사진

2 문제 제기: 왜 이 사업이 필요한가?

🎯 역할: 클라이언트가 겪는 어려움이나 지역사회의 욕구를 진정성 있게 전달하여 공감을 이끌어냅니다.
💡 현장에 맞는 "문제" 예시
  • "학교 졸업 후, 발달장애 청년 10명 중 7명은 갈 곳이 없습니다."
  • "지류 안내문만으로는 시각장애인 분들이 정보를 얻기 어렵습니다." (웹 접근성 확보 필요성)

3 해결 제시: 기관의 전문적인 개입

🎯 역할: 제기된 문제(욕구)를 우리 복지관/프로그램이 어떻게 해결할 것인지 구체적으로 보여줍니다.
💡 사업 안내 예시
  • 해결책: "1:1 맞춤형 직무지도원 배치와 현장 실습 지원"
  • 프로그램 내용: 1) 기초 소양 교육 → 2) 카페 실무 실습 → 3) 취업 연계

4 신뢰 요소: 투명성과 참여자 후기

🎯 역할: 사회복지 기관의 생명인 '투명성'과 '전문성'을 입증하여 후원/참여의 장벽을 낮춥니다.
💡 신뢰 구축 예시
  • "작년 한 해, 15명의 청년이 취업에 성공했습니다." (구체적 통계)
  • "기부금은 100% 훈련생의 실습 재료비로 사용됩니다." (투명성)
  • "이전 기수 참여자 김OO 님의 변화 스토리" (스토리텔링 후기)

5 CTA: 신청/후원 버튼 (Call To Action)

🎯 역할: 방문자가 구체적인 행동(참여, 기부 등)을 취하도록 돕는 명확한 버튼입니다. 구글 폼이나 후원 결제창으로 연결됩니다.
💡 추천 CTA 조합 (사회복지용)
  • 프로그램: "훈련생 신청서 작성하기"
  • 후원: "월 1만원으로 자립 응원하기"
  • 봉사: "자원봉사 VMS 연계 신청하기"

3 3장. Canva로 디자인 초안 잡기

디자인 뼈대 세우기

Canva의 '웹사이트' 템플릿을 활용해 복지관 랜딩페이지의 구조와 색감을 빠르게 시각화합니다.

  1. 1 Canva 접속 및 로그인
  2. 2 “비영리 웹사이트” 또는 “웹사이트” 검색
  3. 3 5단계 구조에 맞춰 섹션 구성
  4. 4 기관 로고 및 현장 사진 삽입

Canva 디자인을 코드로 변환하는 이유

  • Canva 자체 배포는 구글 폼(설문) 삽입 등 외부 기능 연동에 한계가 있습니다.
  • HTML 코드로 변환(Gemini 활용)하면 시각장애인용 '웹 접근성(스크린리더 읽기)' 코드를 직접 추가할 수 있어 복지 기관에 필수적입니다.

✍ 실습 3: 시각 자료 체크리스트

4 4장. GitHub로 무료 배포하기

4-1. 예산 없는 복지기관의 구원투수, GitHub Pages

GitHub에서 제공하는 서버비/호스팅비 완전 무료 서비스입니다. 예산 품의 없이 담당자 선에서 즉시 페이지를 발행할 수 있습니다.

4-2. 배포 절차 요약

Step 1 GitHub 가입
Step 2 Repository 생성
Step 3 index.html 등 업로드
Step 4 Settings → Pages 배포 설정
Step 5 고유 웹 주소 생성!

✍ 실습 4: 저장 구조 만들기

HTML 파일을 정리할 기본 폴더 구조를 만들어 보세요.

welfare-campaign/
├── index.html # 메인 HTML 파일 (Gemini가 짜준 코드)
├── style.css # 디자인 파일 (Tailwind 쓰면 생략 가능)
└── images/ # 복지관 로고, 프로그램 활동 사진 등

5 5장. Gemini로 HTML 자동 생성하기

5-1. 왜 Gemini인가요?

Google의 최신 AI 모델인 Gemini는 복잡한 코딩을 몰라도, 사회복지사의 '기획 의도'를 설명글(프롬프트)로 적어주면 그것을 실제 동작하는 웹페이지 코드로 완벽하게 번역해 줍니다.

5-2. 복지 현장용 기본 프롬프트(명령어) 템플릿

장애인 복지관 주간보호센터 신규 이용자 모집용 랜딩페이지를 HTML 코드로 만들어줘.
모바일 반응형으로 제작하고, Tailwind CSS를 사용해줘.
시각장애인을 위한 웹 접근성(alt 태그 등)을 철저히 준수해줘.
가장 아래에는 '상담 예약하기' CTA 버튼을 넣어줘.
스크롤형 단일 페이지로 구성해.

6 6장. 응용: 연간 사업안내 랜딩페이지 만들기

책자 형태의 두꺼운 사업안내서 대신, 스마트폰으로 쉽게 보는 인포그래픽 랜딩페이지를 만들어 이해관계자(지자체, 후원자)에게 공유할 수 있습니다.

💡 예시 요청 문구 (프롬프트)

2026년 OO장애인종합복지관 3대 핵심 추진 사업 안내를 인포그래픽 스타일 랜딩페이지로 제작해줘.
아이콘을 적극 활용하고, 색상은 신뢰감을 주는 블루 계열로 해줘.
스마트폰으로 볼 때 가독성이 높도록 폰트 크기를 키워줘.

✔ 최강의 업무 효율 콤보: Canva + Gemini + GitHub

단계 담당 도구 및 역할
1. 기획 및 디자인 초안 Canva (시각화 및 팀 내 보고용)
2. 웹 접근성/기능 코드화 Gemini (HTML/CSS 코드 자동 생성)
3. 예산 0원으로 배포하기 GitHub (서버 호스팅 제공)

7 7장. 현장 사회복지사 적용 전략

바쁜 현장 실무자에 최적화된 활용법

  • ✓
    지류 중심의 관보 및 월간 소식지를 언제든 모바일로 접속하고 글씨 크기 조절이 가능한 랜딩페이지로 전환 (접근성 향상)
  • ✓
    프로그램 모집 포스터 하단에 구글 폼으로 직결되는 QR코드를 연결하여 신청 누락 및 이탈 방지
  • ✓
    특정 타겟을 겨냥한 집중 후원 캠페인 전용 페이지 (예: 휠체어 수리비 지원 프로젝트) 제작으로 모금 효율 극대화
  • ✓
    자원봉사자 모집 및 교육 수합을 자동화하여 실무자의 단순 전화 응대 및 서류 업무 시간 단축
